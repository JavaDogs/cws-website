Who is this for

Although security is the key part of using CWS - it is not possible to achieve a level of security which will be demanded from many corporations, organizations or governments. To obtain such a degree of security, the best advice is to not use hosting or cloud services. As they will inherently be your achilles heel.

CWS is aiming at improving security for others, who neither can nor will pay for additional security by either hosting data themselves or use other means of improved tools. The goal is not to achieve perfect security but to make life harder for hackers to read out data.

If this is what is your goal, then CWS may be a tool which use can find useful in your chain.

Disclaimer

As stated above, CWS neither can nor will aim at perfect security. That said, the work with developing CWS has been with the primary focus on improving backend security, by ensuring that what is stored is all encrypted.

Generally, there are 3 areas where security will come up short:

  • Hosting or Cloud Provider
    If the hosting provider is not taking their customers data security seriously, either by not patching their software or protecting their services using firewalls. Then, CWS cannot give you a guarentee that your data is secure. As CWS requires that the underlying software to run Java & Java EE must be properly updated.
  • Network Communication
    CWS uses WebServices (REST / SOAP) for all requests. this means that the communication must be properly secured with SSL. If not, then all data send or received may be compromised, as it can be sniffed over the net. As SSL is added to the containers, it is not part of CWS, and as with patches & firewalls, CWS relies on a proper setup.
  • Data Processing
    During the processing of data, both the data and the keys will appear in readable form in the memory of the host. It is possible for someone with access to the hosts to make memory dumps, and this way obtain the information.

Of these shortcomings, the last sounds like the worst, but as CWS is suppose to be added between your Persistence Logic & Database, the data is already present in other components and reading it out from those may be simpler. Put in short, the aim is not to focus on security of the processing, but storing - hence the name Crypto Store.
